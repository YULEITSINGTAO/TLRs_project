The Cook Book of Data Processing in TLR Project¶

1. Introduction¶

This document mainly notes how did we do the computational work from the raw dataset downloaded from the Internet. This document denotes how to get the dataset from the website, the data processing steps, how to detect repeat units and some ideas from the result.

In this project, programs 'FFAS' and 'FAIT' are in the cluster: Godziklab/shared/FAIT_1. The document share the code about how to preprocess input data and analysis the results.

2. Obtain Data¶

In this project, we download the TLR and NLR protein sequences from the website: Gnsemblgenomes (http://ensemblgenomes.org/info/genomes?division=2) to get the protein sequences of 'Strongylocentrotus purpuratus'. We download the dataset and name it as 'Strongylocentrotus_purpuratus.Spur_3.1.pep.all.fa'.

3. DATA Processing¶

'Strongylocentrotus_purpuratus.Spur_3.1.pep.all.fa' denotes different kinds of protein sequences in Sea Urchin and we are interested in TLR and NLR proteins. Therefore, we need to select these two kinds of protein from the whole dataset.

For the initial dataset, we need to do several processing steps:

  1. Stelect TLR and NLR from the dataset. (This step is made using bash)
In [ ]:
# !/bin/bash

awk '/^>/ { p = ($0 ~ /NLR/)} p' Strongylocentrotus_purpuratus.Spur_3.1.pep.all.fa > Strongylocentrotus_NLR.fa

echo The number of NLR in Stronglocentrotus is
grep -c "^>" Strongylocentrotus_NLR.fa

Save the bash code as select.bash then run the code and we can get two files which note the TLR sequences and NLR sequences: 'Strongylocentrotus_TLR.fa' and 'Strongylocentrotus_NLR.fa'.

As program 'FAIT' deals with one sequence at one time, we need to export TLR and NLR sequences into individual files (i.e. one sequences in one file): In this step, we can use the function in Python package 'Biopython' and the code is from the website https://www.biostars.org/p/273248/.

In [ ]:
# This code is used to split the multiple sequences into a sequence file.
from Bio import SeqIO
import argparse

parser = argparse.ArgumentParser(description="Split the fasta file into individual file with each gene seq")
parser.add_argument('-f', action='store', dest='fasta_file', help='Input fasta file')
result = parser.parse_args()

f_open = open(result.fasta_file, "rU")

for rec in SeqIO.parse(f_open, "fasta"):
   id = rec.id
   seq = rec.seq
   id_file = open(id, "w")
   id_file.write(">"+str(id)+"\n"+str(seq))
   id_file.close()

f_open.close()

The table four columns: Sequence ID, Units Number, Units Length and Variance. The following code is help us to grab these features and make the table.

path = 'Godziklab/shared/FAIT_1/sequences_1/out/*-tr.txt'   
files=glob.glob(path)   
summary = pd.DataFrame()
for file in files:  
    ID = file[3:len(file)-7]
    f = open(file,'r')  
    s = f.read()
    result = re.findall("UnitLength=\"(\d+)\"",s) 
    result = list(map(int, result))
    variance = round(np.var(result),2)
    unit_number = round(len(result),2)
    average_length = round(sum(result)/len(result),2)
    data = [ID, unit_number, variance, average_length]
    df = pd.DataFrame(data)
    summary = pd.concat([summary,df], axis = 1, ignore_index = True)
       
    f.close()

The idea to show the similarity between each sequence¶

Method I: Pearson Correlation:¶

If we have two sequences: sequence_1 = [24,26,27,23,25]; sequence_2 = [23,24,23].

Then we can use two steps to show their correlation with each other

*Step 1*: Since sequence_1 and sequence_2 have different length, we use 0 to fill the shorter one, which makes sequence be sequence_2 = [23,24,23,0,0]

*Step 2*: We can see the corrleation between two sequences
